# Исправление заданий (Code Style + Ошибки)

В данном репозитории приведены исправленные версии четырёх задач на C++.
Основные цели изменений:

* привести код к более читаемому code-style (близкому к Epic Games style);
* исправить логические и структурные ошибки;
* добавить проверки входных данных;
* улучшить читаемость имён переменных и функций;
* сохранить исходную логику алгоритмов.

Изменения выполнялись только для имён переменных, функций и структур, как требовалось в задании.

# Задание 1 — Артефакты (оптимальный выбор предметов)

## Описание задачи

Дано N артефактов с весами и объёмами.
Необходимо выбрать подмножество предметов так, чтобы:

* суммарный вес находился в диапазоне [A, B]
* суммарный объём был минимальным

Используется динамическое программирование.

## Найденные ошибки

### 1. Отсутствие проверки открытия файла

Исходный код продолжал выполнение даже если файл не открылся.

Было:

if (!InputFile.is_open()) cout << "Такого файла не существует";
Проблема:

* программа продолжает работу с невалидным потоком.

Исправление:

if (!InputFile.is_open())
{
    cout << "Такого файла не существует";
    return;
}
---

### 2. Некорректное сообщение при отсутствии решения

Было:

cout << ":";
Проблема:

* сообщение не информативно.

Исправление:

cout << "Решение отсутствует";
---

### 3. Потенциальная проблема восстановления решения

Используется одномерный массив DP, который не хранит информацию о выбранных предметах.
Это может привести к некорректному восстановлению набора.

В отчёте отмечено, что корректное решение требует хранения таблицы выбора.

---

| Было | Стало  |
| ---- | ------ |
| ci   | Ci     |

| i    | Index  |

| dp   | Dp     |

# Задание 2 — Черепашка (максимальный путь)

## Описание задачи

На квадратной доске N × N черепашка начинает движение из правого верхнего угла
и может двигаться:

* влево
* вниз

Нужно найти максимальную сумму чисел на пути до левого нижнего угла.

Алгоритм решается методом динамического программирования.

---

## Найденные ошибки

### 1. Отсутствие проверки открытия файлов

Исправлена проверка:

if (!InputFile.is_open() || !OutputFile.is_open())
---

### 2. Некорректная проверка диапазона N

По условию:

1 < N < 80
Исправлено условие:

if (N <= 1 || N >= 80)
---

### 3. Плохая читаемость имён

| Было | Стало     |
| ---- | --------- |
| matr | Board     |
| dosk | DpTable   |
| n    | N         |
| res  | ResultSum |
| i    | Row       |
| j    | Col       |

---

### 4. Улучшение структуры программы

Логика программы вынесена в функцию:

MainTurtleTask()
что улучшает читаемость.

# Задание 3 — K-ичные числа

## Описание задачи

Нужно определить количество N-разрядных чисел в системе счисления с основанием K,
в записи которых содержится более трёх подряд идущих нулей.

Используется динамическое программирование по состояниям.

Состояния:

* нет нулей на конце
* один ноль
* два нуля
* три нуля
* четыре и более нулей

---

## Найденные проблемы

### 1. Непонятные имена переменных

Исходный код:

nz, oz, tz, thz, foz
Исправлено:

| Было | Стало               |
| ---- | ------------------- |
| nz   | NoZeroCount         |
| oz   | OneZeroCount        |
| tz   | TwoZeroCount        |
| thz  | ThreeZeroCount      |
| foz  | FourOrMoreZeroCount |

---

### 2. Неясный комментарий

Было:

4 нуля
Исправлено:

четыре и более нулей подряд
Поскольку состояние хранит не только 4, а 4 и более.

---

### 3. Улучшение читаемости цикла

Переменная цикла:

i → DigitIndex
---

# Задание 4 — Разбор логического выражения

## Описание задачи

Дано логическое выражение в виде строки:

T
F
And(expr,expr)
Or(expr,expr)
Необходимо вычислить значение выражения.

Решение реализовано с помощью рекурсивного разбора выражения.

---

## Найденные ошибки

### 1. Использование цикла

В исходном коде был цикл:

while (...)

Проблема:

По условию задачи нельзя использовать оператор цикла.

Исправление:

* цикл удалён
* добавлена рекурсивная проверка строки

---

### 2. Проверка пробелов

По условию строка не должна содержать пробелов.

Добавлена функция:

CheckSpaces()
которая рекурсивно проверяет строку.

Если найден пробел:

программа завершается
---

### 3. Лишнее сравнение с true

Было:

if (res == true)
Исправлено:

if (ResultValue)
---
